# Markdown 阅读器详细技术设计

## 1. 文档目标

本文档用于指导基于 Java 技术栈实现 macOS 轻量化 Markdown 阅读器的第一阶段开发，覆盖以下内容：

1. 技术栈与依赖选择
2. 系统分层与模块职责
3. 核心页面结构与交互流程
4. 关键类设计
5. Markdown 渲染链路
6. 配置、持久化与资源管理
7. 打包发布方案
8. MVP 开发顺序

本文档面向接下来的项目框架搭建与编码实现，目标是让项目从第一天起就保持可维护、可演进。

## 2. 项目边界

### 2.1 项目名称

暂定项目名：`md-reader`

### 2.2 目标平台

- 主目标平台：`macOS`
- 技术路线保留跨平台潜力，但首轮体验以 macOS 可用为准

### 2.3 首版目标

实现一个可运行、可打包、可日常阅读本地 `.md` 文件的桌面应用，支持：

- 打开单个 Markdown 文件
- 渲染标题、段落、列表、引用、链接、表格、代码块、图片
- 左侧目录导航
- 当前文档关键字搜索
- 浅色/深色主题切换
- 字体大小调整
- 最近打开文件记录

## 3. 技术选型

## 3.1 语言与运行时

- 语言：`Java`
- JDK：`JDK 21 LTS` 为目标版本

说明：

- 当前开发机可能装有更高版本 JDK，但项目编译目标建议固定为 `21`
- 这样有利于后续打包与长期兼容

## 3.2 UI 框架

- `JavaFX`

选型原因：

1. 能满足桌面应用需要
2. 自带 `WebView`
3. CSS 能力优于 Swing
4. 适合轻量工具型应用

## 3.3 Markdown 解析库

- `flexmark-java`

使用原因：

1. 对 GFM 常见能力支持较好
2. 扩展灵活
3. 更适合目录提取和 HTML 渲染

## 3.4 构建工具

- `Maven`

使用原因：

1. 结构清晰
2. JavaFX 集成成熟
3. 依赖管理简单
4. 适合当前项目规模

## 3.5 应用打包

- `jpackage`

目标产物：

- `.app`
- 视需要增加 `.dmg`

## 4. 系统总体架构

项目采用轻量桌面应用的分层结构：

1. **表现层（UI）**
   - 窗口布局
   - 工具栏
   - 目录树
   - 阅读区
   - 搜索栏

2. **控制层（Controller）**
   - 接收 UI 事件
   - 调用服务层
   - 更新界面状态

3. **服务层（Service）**
   - 文件读取
   - Markdown 解析
   - 搜索
   - 偏好设置保存
   - 最近文件管理

4. **模型层（Model）**
   - 目录节点
   - 当前文档
   - 阅读器状态
   - 用户设置

5. **基础设施层（Util / Resource）**
   - HTML 模板装配
   - 本地资源路径转换
   - 文件编码处理
   - 路径规范化

## 5. 核心渲染流程

Markdown 阅读器的核心链路如下：

```text
用户选择文件
    ↓
FileManager 读取 Markdown 文本
    ↓
MarkdownService 解析 Markdown AST
    ↓
提取标题结构生成 TOC
    ↓
Markdown 渲染为 HTML 片段
    ↓
HtmlTemplateService 注入 CSS / JS / 主题参数
    ↓
WebView 加载完整 HTML
    ↓
ReaderController 同步目录、标题、状态栏
```

### 5.1 为什么采用 HTML 渲染

不直接使用 JavaFX 原生控件逐段绘制 Markdown，原因如下：

1. 表格、代码块、图片布局实现复杂
2. 主题与样式切换成本高
3. 锚点跳转与目录联动实现不如 HTML 自然
4. 后续导出 PDF 或导出 HTML 更方便

因此首版统一采用：

**Markdown -> HTML -> WebView**

## 6. 界面设计

## 6.1 主窗口结构

主窗口采用 `BorderPane + SplitPane`：

- 顶部：工具栏 `ToolBar`
- 中部：左右分栏 `SplitPane`
  - 左侧：目录树 `TreeView<TocItem>`
  - 右侧：阅读区 `WebView`
- 底部：状态栏 `HBox` 或 `Label`

### 6.2 工具栏控件

建议包含以下控件：

1. `打开文件` 按钮
2. `重新加载` 按钮
3. `搜索框`
4. `搜索上一个 / 下一个`
5. `主题切换` 按钮
6. `字号减小`
7. `字号增大`

### 6.3 左侧目录栏

使用 `TreeView<TocItem>` 展示标题层级：

- `#` 为一级节点
- `##` 为二级节点
- `###` 为三级节点

点击目录项后：

- 调用 WebView 内锚点跳转
- 同步更新当前选中节点

### 6.4 阅读区

阅读区由 `WebView` 承担：

- 加载完整 HTML 页面
- 内置主题 CSS
- 可注入少量 JS 完成定位和搜索高亮

### 6.5 状态栏

建议展示：

- 当前文件名
- 文件路径简写
- 渲染状态
- 当前主题
- 缩放/字号信息

## 7. 模块设计

## 7.1 `App`

职责：

- JavaFX 应用入口
- 初始化主窗口
- 组装控制器与服务
- 处理启动参数（未来可支持 Finder 关联打开）

关键方法：

- `start(Stage stage)`
- `main(String[] args)`

## 7.2 `ReaderController`

职责：

1. 管理主界面组件
2. 响应用户操作
3. 驱动渲染流程
4. 管理当前状态

关键能力：

- 打开文件
- 渲染当前文件
- 刷新内容
- 更新 TOC
- 跳转标题
- 主题切换
- 字号调整
- 搜索触发

建议方法：

- `openFile(Path path)`
- `reloadCurrentFile()`
- `renderDocument(DocumentData data)`
- `applyTheme(Theme theme)`
- `adjustFontSize(int delta)`
- `search(String keyword)`

## 7.3 `FileManager`

职责：

- 打开和读取文件
- 解析文件路径
- 校验扩展名
- 统一字符编码策略

建议方法：

- `readMarkdown(Path path): DocumentSource`
- `isMarkdownFile(Path path): boolean`

设计说明：

- 默认按 UTF-8 读取
- 若需要可后续增加编码自动探测

## 7.4 `MarkdownService`

职责：

- Markdown 解析
- TOC 提取
- HTML 渲染

输入：

- Markdown 原文
- 文档所在目录
- 当前主题与展示配置（部分参数）

输出：

- `RenderedDocument`

建议方法：

- `render(DocumentSource source): RenderedDocument`

### 输出对象 `RenderedDocument`

应包含：

- `title`
- `htmlBody`
- `tocItems`
- `baseDir`
- `sourcePath`

## 7.5 `HtmlTemplateService`

职责：

- 将 HTML 片段组装成完整 HTML 文档
- 注入主题样式与脚本
- 注入基础元信息

建议方法：

- `buildHtml(RenderedDocument document, ReaderPreferences preferences): String`

模板内容应包括：

- `<meta charset="UTF-8">`
- 主题样式
- 代码高亮样式
- 基础脚本（滚动定位、搜索标记）

## 7.6 `ResourceResolver`

职责：

- 将 Markdown 中的相对资源路径转换为 WebView 可访问 URI
- 处理图片、本地链接和锚点链接

关键问题：

- `./img/a.png`
- `../docs/b.md`
- 包含空格和中文路径

建议输出：

- `file:///...` 格式 URI

## 7.7 `SearchService`

职责：

- 协调 WebView 中的关键字查找
- 在后续版本中可扩展为搜索结果计数与定位

首版实现建议：

- 通过注入 JS 在页面中执行简单查找和高亮

## 7.8 `PreferenceService`

职责：

- 保存与读取用户偏好

保存内容：

- 主题
- 字体大小
- 最近打开文件
- 窗口尺寸

存储建议：

- 使用本地 JSON 文件

默认路径建议：

```text
~/Library/Application Support/md-reader/preferences.json
```

## 7.9 `RecentFileService`

职责：

- 管理最近文件列表
- 去重
- 控制数量上限
- 剔除已不存在文件

建议上限：

- 10 或 20 条

## 8. 数据模型设计

## 8.1 `DocumentSource`

表示文件读取后的原始文档：

- `Path path`
- `String rawMarkdown`
- `String fileName`
- `Path baseDir`

## 8.2 `RenderedDocument`

表示 Markdown 渲染后的结果：

- `String title`
- `String htmlBody`
- `List<TocItem> tocItems`
- `Path sourcePath`
- `Path baseDir`

## 8.3 `TocItem`

表示目录节点：

- `String text`
- `String anchorId`
- `int level`
- `List<TocItem> children`

说明：

- UI 中可直接转成 `TreeItem<TocItem>`

## 8.4 `ReaderPreferences`

表示用户偏好：

- `Theme theme`
- `int fontSize`
- `double windowWidth`
- `double windowHeight`
- `List<String> recentFiles`

## 8.5 `ReaderState`

表示当前运行态：

- `Path currentFile`
- `RenderedDocument currentDocument`
- `boolean dirtyView`
- `String currentSearchKeyword`

说明：

- 虽然这是阅读器不是编辑器，仍可保留少量运行态，便于刷新和状态同步

## 9. Markdown 解析策略

## 9.1 支持范围

首版建议支持以下 Markdown 能力：

- 标题
- 段落
- 列表
- 引用
- 分隔线
- 行内代码
- 围栏代码块
- 链接
- 图片
- 表格
- 粗体 / 斜体

## 9.2 标题锚点策略

每个标题生成稳定锚点 `id`：

1. 优先使用解析库自带锚点能力
2. 若自定义生成，需解决重名问题

例：

- `简介` -> `intro`
- 第二个同名标题 -> `intro-2`

中文标题建议不要直接原文做 id，优先做规范化或哈希化，避免兼容性问题。

## 9.3 表格与代码块

通过 `flexmark-java` 扩展支持：

- `TablesExtension`
- `AutolinkExtension`
- `StrikethroughExtension`

必要时再加：

- `TaskListExtension`

## 10. 搜索设计

首版搜索只做“当前文档内搜索”，不做全局文件搜索。

## 10.1 交互流程

1. 用户输入关键字
2. 控制器将关键字传入 WebView
3. WebView 中执行搜索脚本
4. 高亮匹配内容
5. 提供上一个/下一个定位能力

## 10.2 首版简化方案

可先实现：

- 搜索框输入后触发页面内查找
- 若找不到则提示

后续增强：

- 匹配数统计
- 当前匹配序号
- 所有匹配项高亮

---

## 11. 主题与样式设计

## 11.1 主题模型

定义枚举：

- `LIGHT`
- `DARK`
- `SYSTEM`（第二阶段再补）

首版先支持：

- `LIGHT`
- `DARK`

## 11.2 CSS 结构

建议拆成：

- `base.css`
- `light.css`
- `dark.css`

样式覆盖内容：

- 页面背景
- 正文颜色
- 标题层级
- 链接颜色
- 代码块背景
- 表格边框
- 引用块样式
- 图片最大宽度

## 11.3 字号调整

优先通过注入 CSS 变量实现：

```css
:root {
  --reader-font-size: 16px;
}
```

页面正文统一使用：

```css
body {
  font-size: var(--reader-font-size);
}
```

## 12. 配置与持久化设计

## 12.1 配置文件位置

macOS 推荐路径：

```text
~/Library/Application Support/md-reader/preferences.json
```

## 12.2 配置结构示例

```json
{
  "theme": "LIGHT",
  "fontSize": 16,
  "windowWidth": 1280,
  "windowHeight": 820,
  "recentFiles": [
    "/Users/demo/Documents/notes/readme.md"
  ]
}
```

## 12.3 保存时机

建议在以下时机保存：

1. 用户切换主题
2. 用户调整字号
3. 打开新文件后更新最近记录
4. 窗口关闭前保存窗口尺寸

## 13. 异常与错误处理

## 13.1 文件读取失败

提示内容应包含：

- 文件名
- 失败原因

常见原因：

- 文件不存在
- 无权限
- 编码异常

## 13.2 资源加载失败

图片或本地链接失效时：

- 页面内显示占位提示
- 状态栏可提示资源未找到

## 13.3 Markdown 渲染失败

若解析异常：

- 保底显示原始文本或错误说明
- 不让应用直接崩溃

## 16. 打包设计

## 16.1 本地开发运行

优先使用 Maven 插件运行：

```bash
mvn javafx:run
```

## 16.2 发布打包

后续使用 `jpackage` 输出 macOS 应用：

```bash
jpackage \
  --name md-reader \
  --input target \
  --main-jar md-reader.jar \
  --main-class com.example.mdreader.App \
  --type app-image
```

首轮可以先完成运行与 jar 打包，再补齐 `.app` 和签名流程。
